Research Strategy

Approach (Phase I): Workflow, validation plan, and measurable KPIs

Phase I hardens PromptGenix into a reviewer-ready, reproducible BioAI automation platform using public datasets only. The goal is to demonstrate: (i) reliable end-to-end pipelines for RNA-Seq and flow cytometry, and (ii) a traceable Hypothesis Generator that produces ranked, testable hypotheses with PubMed-linked evidence.

1) Phase I system workflow (end-to-end)

  • Input: FASTQ / FCS (or public accession IDs) + minimal metadata template.
  • Automated pipelines: RNA-Seq (QC → alignment/quant → DEG → GSEA/GSVA) and flow cytometry preprocessing (QC → transforms → batch normalization → automated gating/summary).
  • Evidence-aware LLM layer: reads results, pulls supporting biology from PubMed + curated knowledge, then generates structured interpretation.
  • Hypothesis Generator: outputs ranked hypotheses + concrete follow-up experiments; each item includes traceable citations and “why this hypothesis” rationale.
  • Output: exportable HTML/PDF report (methods + figures + key findings + hypotheses + recommended experiments).

2) Validation datasets & study design

  • Public-only evaluation: GEO/SRA for RNA-Seq, FlowRepository for flow cytometry (no restricted cohort data in Phase I).
  • Replicates & reruns: each dataset processed across multiple reruns to quantify reproducibility and stability.
  • Ground-truth anchors: compare recovered signals to known published contrasts (DE genes, pathways, canonical immune signatures).
  • Usability pilot: 2–3 partner reviewers (PI/analyst) score report clarity and hypothesis usefulness using a short rubric.
Selection rule: choose ≥3 datasets spanning disease area and assay variability (different platforms / panels) to stress-test robustness.

3) Phase I success metrics (KPIs) & deliverables

  • KPI-1: Turnaround time — target ≥70% reduction vs. manual analysis baselines (time-to-report).
  • KPI-2: Reproducibility — target ≥90% reproducibility across reruns (config-locked outputs, stable summary statistics).
  • KPI-3: Hypothesis usefulness — target ≥80% “useful/actionable” ratings from pilot rubric.
  • D1: hardened end-to-end pipelines on ≥3 public datasets (RNA-Seq + flow).
  • D2: traceable Hypothesis Generator with ranked hypotheses + experiment suggestions + PubMed citations.
  • D3: reviewer-ready HTML/PDF reports generated in <24 hours from raw inputs.
Note: KPI thresholds are adjustable to solicitation guidance; all Phase I validation uses public datasets to avoid IP/data-use conflicts.
