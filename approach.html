Research Strategy – Approach (Phase I Workflow & Validation)
PromptGenix LLC (Women & Minority-Owned) · Contact: dohoon.kim1@icloud.com · promptgenix.org
Phase I goal

Build a validated prototype that prioritizes testable biological hypotheses using evidence-weighted Bayesian inference over public RNA-Seq / scRNA-Seq and high-dimensional flow cytometry datasets, with strict traceability from ranked outputs back to evidence objects.

Guardrail: Large language models are used only for interpretability (clear “why” explanations + citations), and never to set confidence. Confidence comes from statistical evidence + Bayesian updating.

Workflow overview

  • Inputs: disease / protein / pathway / cell-type question (optionally with accession IDs).
  • Evidence pool: automated retrieval of relevant public datasets + metadata + literature signals.
  • Evidence generation: standardized feature extraction (effect sizes, uncertainty, reproducibility, context).
  • Decision engine: statistical scoring + Bayesian updating to compute posterior confidence distributions.
  • Outputs: ranked hypotheses + confidence bands + evidence coverage (supporting vs. conflicting) + next steps.

Hypothesis prioritization engine (traceable)

  • Candidate hypotheses: structured entities/relations (disease–gene–pathway–cell-type) seeded from public sources.
  • Evidence strength: magnitude + uncertainty + directionality + cross-dataset consistency.
  • Bayesian confidence: priors (biology + context) updated by likelihood from observed evidence.
  • Ranking outputs: posterior mean + credible interval + evidence coverage.
  • Actionability: each hypothesis includes suggested validation experiment(s) and required data.
Phase I emphasis: transparent ranking with uncertainty (not “black-box generation”).
Bayesian evidence-to-confidence mapping

For each hypothesis H (e.g., “Protein X is upregulated in disease Y in cell-type Z”), PromptGenix computes a posterior confidence by combining a structured prior with evidence-derived likelihoods aggregated across datasets.

1) Prior construction (bounded, interpretable)
P(H) = σ( wp·Pathway(H) + wc·CellSpec(H) + wl·LitSupport(H) + b )

2) Likelihood from evidence objects (per dataset i)
Ei = { effect size di, SEi, pi, direction, QC, context }
log BFi ≈ κ · sign(di) · |di| / (SEi + ε)   (monotone in magnitude; penalizes uncertainty)

3) Cross-dataset aggregation with consistency weighting
log BF = Σi αi · log BFi,   αi ∝ QCi · CohortMatchi · Consistencyi

4) Posterior (reported as mean + credible interval)
Posterior odds: O(H|E) = O(H) · BF
P(H|E) = O(H|E) / (1 + O(H|E)),   where O(H)=P(H)/(1-P(H))
Validation plan & success metrics
  • Feasibility: end-to-end accession-driven runs produce deterministic artifacts (versioned configs + checksums).
  • Traceability: each ranked hypothesis links to evidence objects (supporting vs. conflicting) and citations.
  • Performance target: generate reviewer-ready HTML/PDF report within <24 hours (dataset-dependent).
  • Utility: pilot-style evaluation (2–3 public-data studies): time-to-hypothesis reduced + usefulness rating.
