Research Strategy – Approach (Phase I Workflow & Validation)

PromptGenix LLC · Contact: dohoon.kim1@icloud.com · promptgenix.org
Phase I goal

Build a validated, reproducible prototype that produces pipeline results and ranked hypotheses from public datasets.

Phase I focuses on engineering robustness, reproducibility, and measurable performance gains. Validation will rely exclusively on public datasets (e.g., GEO/SRA for RNA-Seq and FlowRepository for flow cytometry), minimizing IP or data-use conflicts while enabling transparent evaluation.

Workflow overview

  • Inputs: FASTQ (bulk or single-cell), FCS files, or public accession IDs.
  • Automated pipelines: QC → preprocessing → analysis (DEG, signatures, enrichment / gating / subsets).
  • Outputs: versioned results + reviewer-ready HTML/PDF report + structured artifacts for reuse.
  • LLM layer: converts results into mechanistic narratives under strict traceability constraints, requiring explicit linkage to observed pipeline outputs and cited literature.
Reproducible configs Versioned outputs HTML/PDF reports Traceable citations

Hypothesis Generator (traceable)

The Hypothesis Generator is designed to produce testable and ranked hypotheses grounded in literature-aware reasoning.

  • Evidence stitching: connect observed signals (DE genes, pathways, immune subsets) to plausible mechanisms.
  • Ranking: prioritize by consistency with dataset signals, coherence with known biology, and actionability.
  • Experiments: recommend concrete next steps (flow panels, contrasts, subset analyses, validation assays).
  • Traceability: include PubMed-style citations and “why this hypothesis” rationale for reviewer transparency.
Phase I emphasis: traceable reasoning over “black-box” generation.
Phase I validation plan

Public-data evaluation with measurable KPIs.

Work package What we will do Success metrics (examples)
WP1
Pipelines		 Harden automated RNA-Seq and flow cytometry pipelines across multiple public datasets. Standardize configuration, logging, and export formats to enable deterministic reruns. ≥90% reproducibility across reruns (same inputs/configs).
≥70% reduction in turnaround time vs. manual workflow baseline.

Hypotheses		 Implement Hypothesis Generator producing ranked hypotheses with supporting evidence from pipeline outputs and traceable literature-aware citations. ≥80% “useful” rating by pilot users, defined as hypotheses that directly inform at least one concrete follow-up analysis or experimental decision.
Clear traceability: each hypothesis includes evidence + citations.

Reduced manual editing time (tracked in pilot feedback).
Data-use boundary: Phase I evaluation uses public datasets only (GEO/SRA/FlowRepository), enabling clean SBIR review and minimizing IP concerns.
Risks & Mitigation

Practical risks addressed with explicit fallback strategies.

Technical risks

  • Dataset heterogeneity: inconsistent metadata / formats across repositories.
  • Compute variability: differences between local Mac Studio and cloud environments.
  • Model drift / hallucinations: LLM may overstate evidence without guardrails.

Mitigation strategies

  • Robust ingest layer: schema detection + validation + standardized output templates.
  • Deterministic runs: pinned versions, config snapshots, and re-run checksums.
  • Traceability guardrails: require citations/evidence per hypothesis; label uncertainty explicitly.
  • Fallback mode: if citations are weak, generate “candidate hypotheses” with a clear confidence flag and recommended data needs.
Reviewer-friendly principle: Every claim must be traceable to (1) observed data outputs or (2) cited literature, with uncertainty explicitly stated.
